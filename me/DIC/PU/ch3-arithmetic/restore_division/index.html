    Restore division

    为了让你彻底搞懂这两个算法在硬件中的具体运转流程，我们选择计算机组成原理（COA）教科书中最经典的无符号二进制除法作为例子：

    计算：\(11 \div 3\)（即 \(11\) 除以 \(3\)）
    * 被除数 (Dividend)：\(Q = 1011\) （共 4 位，\(N=4\)）
    * 除数 (Divisor)：\(M = 0011\) （使用 5 位补码表示为 00011，用于计算。它的求补码减法器输入 \(-M = 11101\)）
    * 累加器 (Accumulator)：\(A = 00000\) （初始化为 0，使用 5 位宽以保留符号位，最高位为符号位）
    * 预期结果：商 \(Q = 3\)（二进制 0011），余数 \(A = 2\)（二进制 00010）。

    下面我们分别用恢复余数法和不恢复余数法完整走一遍这 4 步迭代。

    一、 恢复余数法 (Restoring Division) 完整追踪¶

    恢复余数法的硬件规则：
    1. \((A, Q)\) 整体左移 1 位。
    2. 计算 \(A = A - M\)。
    3. 检查 \(A\) 的符号位：
    * 若 \(A < 0\)（符号位为 1），代表减法失败：商的最低位置 0（\(Q[0]=0\)），并把 \(A\) 加上 \(M\) 恢复原值。
    * 若 \(A \ge 0\)（符号位为 0），代表减法成功：商的最低位置 1（\(Q[0]=1\)），不恢复。

    详细计算步骤：¶

    步骤 操作 累加器 \(A\) 商 \(Q\) 说明
    初始 初始化寄存器 00000 1011 准备开始，计数器 \(Count=4\)
    第 1 步 1. \(AQ\) 整体左移 00001 011_ 被除数最高位进到 \(A\) 的最低位
    2. 减去 \(M\)（\(A = A - M\)） 11110 011_ 00001 + 11101 = 11110
    3. 检查符号位 11110 0110 \(A\) 最高位为 1（负数），所以 \(Q[0]=0\)
    4. 恢复余数（\(A = A + M\)） 00001 0110 11110 + 00011 = 00001（多做了一次加法）
    第 2 步 1. \(AQ\) 整体左移 00010 110_
    2. 减去 \(M\)（\(A = A - M\)） 11111 110_ 00010 + 11101 = 11111
    3. 检查符号位 11111 1100 \(A\) 最高位为 1（负数），所以 \(Q[0]=0\)
    4. 恢复余数（\(A = A + M\)） 00010 1100 11111 + 00011 = 00010（又多做了一次加法）
    第 3 步 1. \(AQ\) 整体左移 00101 100_
    2. 减去 \(M\)（\(A = A - M\)） 00010 100_ 00101 + 11101 = 00010
    3. 检查符号位 00010 1001 \(A\) 最高位为 0（正数），所以 \(Q[0]=1\)，不恢复
    第 4 步 1. \(AQ\) 整体左移 00101 001_
    2. 减去 \(M\)（\(A = A - M\)） 00010 001_ 00101 + 11101 = 00010
    3. 检查符号位 00010 0011 \(A\) 最高位为 0（正数），所以 \(Q[0]=1\)，不恢复
    • 最终结果：
      • 商 \(Q = 0011\)（十进制 \(3\)）
      • 余数 \(A = 00010\)（十进制 \(2\)）
    • 代价分析：在 4 步里，我们一共做了 4 次减法 + 2 次恢复加法 = 6 次加减法操作。

    二、 不恢复余数法 (Non-restoring Division) 完整追踪¶

    不恢复余数法的硬件规则（第 \(i\) 步）：
    1. \(AQ\) 整体左移 1 位。
    2. 检查上一步计算完后 \(A\) 的符号（第一步时初始 \(A \ge 0\)）：
    * 若上一步 \(A \ge 0\)：进行 \(A = A - M\)。
    * 若上一步 \(A < 0\)：进行 \(A = A + M\)。
    3. 检查当前计算完后 \(A\) 的新符号位：
    * 若新 \(A \ge 0\)：置 \(Q[0] = 1\)。
    * 若新 \(A < 0\)：置 \(Q[0] = 0\)。
    4. 特殊修正步：当所有 \(N\) 步迭代完成后，如果最终的 \(A < 0\)，我们需要在最后补做一次 \(A = A + M\) 来纠正最终的余数。

    详细计算步骤：¶

    步骤 操作 累加器 \(A\) 商 \(Q\) 说明
    初始 初始化寄存器 00000 1011 准备开始，此时上一步 \(A \ge 0\)
    第 1 步 1. \(AQ\) 整体左移 00001 011_
    2. 做运算（上一步 \(A \ge 0\) \(\rightarrow\) 做减法） 11110 011_ \(A = 00001 - M = 11110\)
    3. 检查当前符号位 11110 0110 新 \(A\) 最高位为 1（负数），所以 \(Q[0]=0\)
    第 2 步 1. \(AQ\) 整体左移 11100 110_ \(AQ\) 左移，商的 0 移入 \(A\) 尾部
    2. 做运算（上一步 \(A < 0\) \(\rightarrow\) 做加法） 11111 110_ \(A = 11100 + M = 11111\)
    3. 检查当前符号位 11111 1100 新 \(A\) 最高位为 1（负数），所以 \(Q[0]=0\)
    第 3 步 1. \(AQ\) 整体左移 11111 100_ \(AQ\) 左移，商的 0 移入 \(A\) 尾部
    2. 做运算（上一步 \(A < 0\) \(\rightarrow\) 做加法） 00010 100_ \(A = 11111 + M = 00010\)
    3. 检查当前符号位 00010 1001 新 \(A\) 最高位为 0（正数），所以 \(Q[0]=1\)
    第 4 步 1. \(AQ\) 整体左移 00101 001_ \(AQ\) 左移，商的 1 移入 \(A\) 尾部
    2. 做运算（上一步 \(A \ge 0\) \(\rightarrow\) 做减法） 00010 001_ \(A = 00101 - M = 00010\)
    3. 检查当前符号位 00010 0011 新 \(A\) 最高位为 0（正数），所以 \(Q[0]=1\)
    修正 检查最终余数 \(A\) 00010 0011 最终 \(A \ge 0\)，无需任何修正
    • 最终结果：
      • 商 \(Q = 0011\)（十进制 \(3\)）
      • 余数 \(A = 00010\)（十进制 \(2\)）
    • 代价分析：在整个 4 步中，我们一共做了 2 次减法 + 2 次加法 = 4 次加减法操作。

    三、 总结对比（考研/期末高频分析）¶

    通过这个完整的例子，我们可以清晰地看到不恢复余数法的强大之处：

    1. 运算次数减少：
      • 对于同样的 \(11 \div 3\)，恢复余数法做了 6 次算术运算（因为有 2 步失败了，必须当场加回来）。
      • 不恢复余数法只做了 4 次算术运算，整整减少了 33% 的算术开销。
    2. 没有中间的控制气泡：
      • 恢复余数法中，第 1 步和第 2 步都需要“先减，判断，再加”的串行逻辑，这对于流水线设计是灾难性的。
      • 不恢复余数法中，每一步都是“左移，运算，打标签”。运算是加还是减，直接由上一步的结果符号决定。每一步的控制逻辑都是单一、平等的，没有任何回溯（Backtracking）操作。

    这个完整的 Trace 过程是否帮你把“不恢复余数法”彻底理顺了？我们接下来去看看《通用及图形处理器架构》的下一个知识点，比如存储层次结构（Memory Hierarchy）里的 Cache 映射和一致性，还是进入五级流水线（Pipeline）？
