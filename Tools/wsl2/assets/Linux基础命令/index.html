    Linux 基础命令¶

    I. 目录导航 (定位与切换)¶

    1.1 pwd (Print Working Directory)¶

    作用：查看自己当前所在的位置。

    • pwd：直接输出当前所在的完整绝对路径。

    1.2 cd (Change Directory)¶

    作用：在不同目录之间跳转。

    • cd /path/to/dir：进入指定的绝对路径或相对路径。
    • cd ..：返回上一级目录。
    • cd ~：直接跳转回当前用户的家目录 (Home)。
    • cd -：在最近停留过的两个目录之间来回切换（类似电视遥控器的“返回/交替”键，非常实用）。
    • cd /：直接跳转到根目录（整个文件系统的顶层）。

    1.3 ls (List)¶

    作用：查看目录里都有什么。

    • ls：仅列出当前目录下的文件和文件夹名称。
    • ls -l：长格式显示（包含权限、所有者、文件大小、修改时间等详细信息）。
    • ls -lh：最常用。长格式显示，且大小转换为人易读的格式（如 KB, MB, GB）。
    • ls -a：显示所有文件，包括以 . 开头的隐藏文件。
    • ls -lha：王牌组合。详细显示当前目录下所有文件（含隐藏文件）及其易读的大小。
    • ls -lt：按修改时间降序排列（最近修改的文件排在最上面）。
    • ls -ltr：按修改时间升序排列（最近修改的文件排在最下面，看最新生成的日志时常用）。

    II. 文件的增删改查 (CRUD)¶

    2.1 mkdir (Make Directory)¶

    作用：创建新文件夹。

    • mkdir folder_name：在当前目录下创建一个新文件夹。
    • mkdir -p a/b/c：最常用组合。递归创建多级目录。如果父目录 a 或 b 不存在，系统会自动帮你建好，不会报错。

    2.2 touch¶

    作用：创建空文件 / 刷新文件时间。

    • touch file.txt：如果文件不存在，创建一个空的 file.txt；如果文件已存在，不改变其内容，仅把它的“修改时间”更新为当前时间。

    2.3 cp (Copy)¶

    作用：复制文件或文件夹。

    • cp file.txt /path/to/dest/：将普通文件复制到目标目录。
    • cp -r folder/ /path/to/dest/：最常用组合。递归复制整个文件夹及其内部所有内容（复制文件夹必须加 -r）。
    • cp -i file.txt dest/：安全模式。如果目标目录已有同名文件，会在覆盖前提示你确认。
    • cp -p file.txt dest/：完美复制。保留原文件的修改时间、所有者、权限等属性。

    2.4 mv (Move)¶

    作用：移动文件，或者重命名文件。

    • mv file.txt /path/to/dest/：将文件移动到目标文件夹中。
    • mv old_name.txt new_name.txt：重命名。在同一个目录下移动文件，相当于改名。
    • mv -i file.txt dest/：安全模式。目标路径有同名文件时，提示是否覆盖。

    2.5 rm (Remove)¶

    作用：删除文件或文件夹。（注意：Linux 没有回收站，删除即永久丢失）

    • rm file.txt：删除一个普通文件。
    • rm -i file.txt：删除前要求确认（输入 y 或 n）。
    • rm -r folder/：递归删除一个文件夹及其内部所有内容。
    • rm -f file.txt：强制删除（Force），即使文件是只读的，也不会有任何报错或提示。
    • rm -rf folder/：王牌/高危组合。强制且无提示地删除整个文件夹及其内部所有内容（操作前务必确认路径是否正确）。

    2.6 find¶

    作用：在茫茫文件海中搜索特定文件。

    • find . -name "*.py"：最常用组合。在当前目录及其所有子目录下，按名称查找所有以 .py 结尾的文件。
    • find . -name "*test*" -type f：查找名字包含 test 的普通文件（-type d 则是专门找文件夹）。
    • find . -maxdepth 1 -name "*.log"：仅在当前层级（深度为1）查找，不钻进子目录去找。
    • find . -size +100M：查找当前目录下所有大于 100MB 的文件（清理磁盘空间时很好用）。

    III. 文本查看 (从头看、从尾看、翻页看)¶

    3.1 cat (Concatenate)¶

    作用：直接把文件内容全部打印到终端屏幕上。（适合查看几十行以内的小文件）

    • cat file.txt：直接输出文件的所有内容。
    • cat -n file.txt：输出内容，并给每一行加上行号（找代码报错行数时很有用）。
    • cat file1.txt file2.txt > merged.txt：将两个文件的内容合并，并写入到 merged.txt 中。

    3.2 less¶

    作用：分页查看大文件（如几百MB的 EDA 综合日志、DL 训练日志）。它不会一次性把文件全读进内存，所以打开速度极快且不卡顿。

    • less file.log：进入交互式分页查看界面。
      • less 界面内的常用快捷键（必记）：
        • 空格键 或 f：向下翻一页 (forward)。
        • b：向上翻一页 (backward)。
        • /关键字：向下搜索关键字（例如输入 /Error 然后回车）。
        • ?关键字：向上搜索关键字。
        • n：跳到搜索的下一个匹配项。
        • N：跳到搜索的上一个匹配项。
        • q：退出查看 (quit)。
    • less -S file.log：不换行显示。如果日志单行极长，用这个参数可以保持界面整洁（用左右方向键横向滚动）。

    3.3 head¶

    作用：只查看文件的开头部分。

    • head file.txt：默认查看文件的前 10 行。
    • head -n 20 file.txt：查看文件的前 20 行（常用于快速查看数据文件的前几行表头格式）。

    3.4 tail¶

    作用：只查看文件的结尾部分，或者实时监控文件更新。

    • tail file.txt：默认查看文件的最后 10 行。
    • tail -n 50 file.txt：查看文件的最后 50 行。
    • tail -f file.log：王牌组合 (Follow)。实时滚动显示文件的最新追加内容。你跑在后台的程序只要一打印新日志，屏幕就会立刻刷新。（按 Ctrl + C 退出）。

    IV. 文本过滤与统计 (找报错、看数量)¶

    4.1 grep (Global Regular Expression Print)¶

    作用：强大的文本搜索工具，能在文件里精准抓取包含某个关键词的一整行内容。

    • grep "Error" file.log：在文件中寻找包含 "Error" 的行，并打印出来。
    • grep -i "error" file.log：忽略大小写搜索（"Error", "ERROR", "error" 都会被匹配）。
    • grep -n "Error" file.log：显示匹配结果所在的行号。
    • grep -v "Warning" file.log：反向过滤（极度常用）。打印出不包含 "Warning" 的所有行（常用于过滤掉垃圾报警信息，让真正的报错显露出来）。
    • grep -r "main" .：在当前目录及所有子目录的每一个文件里，搜索包含 "main" 的内容。
    • grep -C 3 "Error" file.log：上下文抓取（神器）。不仅显示包含 "Error" 的那一行，还把报错前后的 3行 (Context) 一并打印出来，方便看代码执行的回溯上下文（Traceback）。

    4.2 wc (Word Count)¶

    作用：统计文本的行数、字数、字节数。

    • wc -l file.txt：最常用组合。只统计并输出该文件的总行数 (lines)。
    • wc -w file.txt：统计单词数 (words)。
    • wc -c file.txt：统计字节数 (bytes)。

    补充：黄金搭档 —— “管道符 |”¶

    在文本处理时，| (键盘回车键上方的竖线) 是必须理解的概念。它的作用是：把左边命令的输出结果，直接当成右边命令的输入。

    • cat run.log | grep "Error"
      • 效果等同于 grep "Error" run.log。先读取日志，然后把内容送给 grep 过滤。
    • ls -l | grep ".py"
      • 列出当前目录下所有文件详细信息，但只过滤出名字带 .py 的行。
    • ls -l | wc -l
      • 列出当前目录的文件，并用 wc -l 统计有多少行（也就是统计当前目录下大概有多少个文件+文件夹）。
    • find . -type f | wc -l
      • 在当前目录及子目录下查找所有普通文件，并统计总数（非常实用的命令组合，能快速告诉你一个项目里到底有多少个文件）。
