## Some random thoughts

发现在写处理器代码时才能发现一些平时上课没有注意到的 RISC-V “**小巧思**”

<!-- more -->

不仅仅是不同指令编码格式非常规整，还有

比如这个：

```markdown
### SmlCoke Q

很好，我明白了你给的这段代码的精髓了，好就好在 RISC-V 指令集把所有立即数的最高位都放在了指令的同一位置：最高位，很方便的就可以执行符号位扩展。
不过我还有一个问题，就是 RV32I 指令集中的所有立即数都是有符号数吗？我看你对所有立即数都执行了符号位扩展？

### Gemini 3.1 Pro Preview

你的观察非常敏锐！这正是 RISC-V 指令集架构（ISA）极力追求极简和规整的体现：**指令的第 31 位始终是立即数的符号位，这样硬件可以直接无脑拉出这一根线去进行后续的符号位拷贝扩展（Sign-Extension），大大缩短了关键路径（Critical Path）的延迟。**
```
