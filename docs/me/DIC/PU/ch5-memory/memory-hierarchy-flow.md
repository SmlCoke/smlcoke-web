# Memory Hierarchy Flow（内存层次结构流程） 

这是整个计算机体系结构中**最核心、最宏大的一张运行版图**。我们假设 CPU 现在发出了一条指令要访问内存（比如执行了一次读操作）。下面是这一粒微小的“地址电信号”，在庞大的存储帝国中经历的所有可能旅程：

---

### 第一阶段：地址翻译（从“幻觉”走向“现实”）

**起点：CPU 产生虚拟地址 (VA)。**
硬件首先将虚拟地址拆分为：**VPN (虚拟页号) + Page Offset (页内偏移)**。

#### 1. TLB 查找（闪电战，绝大多数情况）
*   **【TLB 命中 (TLB Hit)】**：
    *   太好了！直接在 TLB 中找到了对应的 **PPN (物理页号)** [1.2.4]。
    *   硬件直接将 **PPN + Page Offset** 拼接，瞬间合成出**物理地址 (PA)** [1.2.9]。
    *   **直接前往【第二阶段：数据访问】** [1.1.2]。
*   **【TLB 缺失 (TLB Miss)】**：
    *   糟糕，TLB 里没有这个翻译关系，硬件不得不走慢速通道 [1.2.6]。
    *   **前往第 2 步**。

#### 2. Page Table 查找（常规通道）
*   MMU 顺着 **PTBR（页表基址寄存器）** 的导航，去物理内存（DRAM）中查该程序的 **Page Table（页表）** [1.2.9]。
*   **【页表命中 (PTE Valid = 1)】**：
    *   找到了！数据其实在内存里，只是刚才没缓存进 TLB [1.1.2, 1.2.9]。
    *   MMU 获取到 **PPN**，并把这个翻译关系**写进 TLB**（以防下次再 Miss） [1.2.9]。
    *   拼接出**物理地址 (PA)** [1.1.2, 1.2.9]。
    *   **前往【第二阶段：数据访问】** [1.1.2]。
*   **【缺页异常 (PTE Valid = 0 / Page Fault)】**：
    *   大事不妙，数据压根不在内存里，还在硬盘上！ [1.2.9]
    *   **前往第 3 步** [1.2.9]。

#### 3. 缺页处理（操作系统介入，最慢通道）
*   CPU 触发 **Page Fault 异常**，暂停当前程序，操作系统（OS）接管 CPU [1.2.9]。
*   OS 去**硬盘 (Disk)** 中定位该页的数据。
*   OS 查看物理内存（DRAM）是否还有空闲位置：
    *   *如果没有空闲物理页*：OS 必须挑一个**牺牲页（Victim Page）**踢走。如果这个牺牲页是脏的（Dirty = 1），必须**将它写回硬盘（Disk）**；然后再把这个物理页腾空。
*   OS 启动 DMA，把新页的数据从**硬盘 (Disk) 读入物理内存 (DRAM)**。
*   OS **修改页表**（将这一页的 Valid 置 1，写入物理页号 PPN），并**更新 TLB**。
*   OS 恢复程序执行，重新发送刚才的虚拟地址。
*   **重新回到第 1 步**（这次一定会进入 【TLB Hit】 的快车道）。

---

### 第二阶段：数据访问（拿到真正的物理地址后）

**此时：MMU 已经拼装出了绝对真实的物理地址 (PA)。**
硬件将物理地址拆分为：**Tag (标记) + Index (索引) + Block Offset (块内偏移)**。

#### 4. Cache 查找（高速通道）
*   硬件根据 Index 找到对应的 Cache Set（组），并在该组内比对 Tag [1.1.2]。
*   **【Cache 命中 (Cache Hit)】**：
    *   狂喜！数据就在 Cache 里。
    *   如果是**读**：Cache 芯片直接吐出数据，送入 CPU 寄存器，**整个访存流程完美结束**。
    *   如果是**写**：根据写策略（Write-Through 或 Write-Back）修改数据 [14, 15]。
*   **【Cache 缺失 (Cache Miss)】**：
    *   没中，只能去主存进货了 [1.1.2]。
    *   **前往第 5 步**。

#### 5. DRAM 访问（最终存储）
*   系统总线向**物理内存 (DRAM)** 发出读请求，读取包含该物理地址的**整个数据块 (Block)** [1.1.2]。
*   **Cache 替换（踢人环节）**：
    *   Cache 必须挑一个 Block 踢走以容纳新块。
    *   如果被踢走的 Block 是脏的（Dirty = 1），必须**将它写回主存 (DRAM)** [15]。
*   将从 DRAM 读回的新 Block **写入 Cache**，同时提取出 CPU 需要的那部分数据，送入 CPU 寄存器。
*   **整个访存流程结束！**
